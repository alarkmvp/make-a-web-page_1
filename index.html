1.1 The Basics of the Web and HTML

Glossary

WWW: World Wide Web, a collection of HTML documents.

HTML: HyperText Markup Language.

URL: Uniform Resource Locator.

HTTP: Hypertext Transfer Protocol.

Web Applications: Is any computer program that runs in a web browser.

Basic understanding of how the internet works

The World Wide Web is a colection of HTML documents.

The process to have access to all this documents starts with the user, a computer and a web browser. The user's computer sends a request through the internet, according to an HTTP protocol, to the servers where the web pages are stored. Then the servers respond, using the same protocol, sending to the users the files they were looking for.

HTML Markups

A HTML markup refers to the sequence of characters or other symbols - code -that you insert at certain places in a text to indicate how it should look when it is displayed. This kind of code, the one used in HTML to specify the formatting, are called tags.

A generic HTML element is structured as follows:

<tag attribute="value"> content </tag>

Computers Are "Stupid"

Computers aren't really "stupids". They simply do things just the way we say. In other words, they follow our instructions. So, if we give them an order in the wrong form - the wrong piece of code - they simply will do it. The computer probably won't notice our mistake.

1.2 Creating a Structured Document

Glossary

HTML: HyperText Markup Language - the standard markup language used to create web pages.

CSS: Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

DOM: Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

HTML, CSS and Javascript

HTML and CSS are both "languages". HTML controls the "structure" of a web page and CSS controls the "style" (how it looks).

Also, There is a third "language" kown as Javascript and is used to program the behavior of the web page.

If we were thinking this as a house: HTML would be the structure (walls, beams, columns); CSS the finishes (floors, paint, ceilings); and Javascript the way in which an interactive element acts (how works: a door, a light or even a tv).

Structured Document

The previous programming languages must follow a specific syntax and rules.

"When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page."

It is named a "tree structure" because the structure of the HTML code resembles a tree.

A generic HTML document is structured as follows:

<!DOCTYPE HTML>

<html;>

	<head>
		<title> title </title>
	</head>

	<body>
		content
	</body>

<html>

1.3 Addings CSS Style and HTML Structure

Glossary

Avoid Repetition

Avoiding repetition will make our lives easier.

We have to identify similar things we want to make or similar problems we need to solve and then write just one piece of code to do it. It will minimize the possibilities of making any kind of mistakes and will help us when we want to make some change, because we will have to make the adjustments in just one place.

Adding CSS Style

CSS is kind of a search and replace tool. Within the CSS file we define rules - styles - that will be applied to the different elements inside the HTML document. We group this elements using class selectors or their tags. That way, we can define display properties like: fonts, margins or colours.

The web browser reads the HTML code and then match the properties found inside of the CSS code to know how to display all the differents elements.

An important thing to be aware is the fact that the rules we define are applied, in a sort of "cascade", from the parent to the child elements. But always the more specific rule is the one taking effect.

The Box Model

Everything inside an HTML document is a BOX. There are boxes everywhere.

The box model is composed by 4 different parts: content - the content of the box, where text and images appear; padding - clears a transparent area around the content; border - a border that goes around the padding and content; and margin - Clears a transparent area outside the border.

Code, Test and Refine

When you define the structure of your webpage, you will need to choose the types of elements you are going to use, the sizes of their boxes and the positions they will have.

So, look for natural boxes. Then look for repeated styles and semantic elements. Start writing your HTML. Then apply styles from biggest to smallest. At last, fix things.

Repeat the process - Code, Test and Refine - until you are satisfied with the results.
